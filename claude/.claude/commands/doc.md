# Document

Generate documentation for: $ARGUMENTS

1. Read the code thoroughly
2. Generate appropriate documentation:
   - For functions: docstrings with params, returns, raises, examples
   - For modules: module-level docstring explaining purpose
   - For classes: class docstring with attributes and methods
   - For APIs: OpenAPI/Swagger spec or README section
   - For projects: README.md with setup, usage, and examples
3. Follow existing documentation style in the codebase
4. Keep it concise but complete

Documentation should help future-you (or Claude) understand the code quickly.
